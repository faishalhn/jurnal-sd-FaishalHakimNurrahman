# Mid-Term Reflection

## Refleksi Pembelajaran Minggu 1–7

Selama tujuh minggu pertama mata kuliah Software Development, saya mempelajari dasar pengembangan perangkat lunak secara kolaboratif, penggunaan Git dan GitHub, manajemen branch, penyelesaian konflik, penerapan arsitektur MVC, serta penyusunan dokumentasi seperti API Contract dan Database Schema.

Materi yang paling sulit saya kuasai adalah Branching, Merging, dan Conflict Resolution. Pada awalnya saya sering bingung ketika harus menggabungkan beberapa branch dan menentukan perubahan mana yang harus dipertahankan saat terjadi konflik. Saya juga mengalami kesulitan memahami alur kerja Git dalam proyek yang melibatkan banyak branch.

Untuk mengatasi kesulitan tersebut, saya berlatih membuat branch baru, melakukan commit secara terpisah, melakukan merge antar branch, serta mencoba menyelesaikan konflik secara langsung. Selain itu, saya mempelajari kembali dokumentasi Git dan mengulangi praktikum hingga memahami hubungan antara branch, commit, merge, dan repository remote.

Dari proses tersebut saya memahami bahwa Git bukan hanya alat penyimpanan kode, tetapi juga sarana kolaborasi yang sangat penting dalam pengembangan perangkat lunak. Saya juga menyadari pentingnya dokumentasi yang baik, commit yang informatif, dan penggunaan workflow yang terstruktur agar proyek dapat berkembang dengan lebih mudah dan terorganisir.

Ke depannya saya ingin meningkatkan kemampuan dalam kolaborasi tim, pengelolaan repository, serta implementasi MVC dan API agar lebih siap menghadapi fase pengembangan perangkat lunak setelah UTS.
